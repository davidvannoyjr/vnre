# 01 Projects — Active Work

Active work, one folder per project. Every project carries a `CLAUDE.md` brief
(one-line purpose · cadence · inputs · outputs · process · status). New project =
duplicate an existing brief, rename, fill in. Deliverables save inside the project
folder — never loose at root. Finished projects move whole to `03 Archive/`.

This tier is the **Projects** leg of the Internal Operating System (Knowledge /
Skills / Projects). Knowledge = `claude/` (the brain + `02-reference/`). Skills =
`tools/`. Projects = here.

## The four-project build (2026-06-23)

Framework prepared for four builds. Each has a brief here; the skills/artifacts
live in `tools/` or their own top-level folder, linked from the brief.

| # | Project | Brief | Build lives in |
|---|---|---|---|
| 1 | **Board of Advisors** — clone the expertise of people DVN trusts into a queryable decision panel | [`board-of-advisors/CLAUDE.md`](board-of-advisors/CLAUDE.md) | `tools/shared/board-of-advisors-skill/` |
| 2 | **Niched Command Center** — one morning screen that answers "what do I do today" | [`niched-command-center/CLAUDE.md`](niched-command-center/CLAUDE.md) | `command-center/` |
| 3 | **AI-Optimized Website** — personal site built for humans *and* AI search, with an "Ask AI about me" block | [`ai-optimized-website/CLAUDE.md`](ai-optimized-website/CLAUDE.md) | `website/` |
| 4 | **Internal Operating System** — ties Knowledge + Skills + Projects together; the `/improve-system` feedback loop | [`internal-operating-system/CLAUDE.md`](internal-operating-system/CLAUDE.md) | `tools/shared/improve-system-skill/` |

Build order is deliberate: **4 frames the house** (the Knowledge/Skills/Projects
model + the improvement loop), **1 and 2 are the daily-use rooms** (decisions and
execution), **3 is the storefront** (how the outside world and AI agents see DVN).
